### Professional Self-Assessment:
This journey of Computer Science program was challenging since I had no knowledge of it along being a deaf single mom of two young kids and working a full time job but all the hard work is totally worth it. I had worked for Walmart company since 2015. After a decade of being in retail career, I noticed that I wanted something different. The feeling of wanting a change had made me researched my options which I found out that as long as I work for Walmart company, my education would be paid. So, I saw that this is my opportunity to achieve a new career! I always loved video games, technology, electronics, computers, so much more! Plus, with our technology thriving through the years, I predict that one of these years in our world will have robots overtaking our jobs and leave all of us jobless. So, I wanted a career that fits my love for technology and electronics along with making sure that my job would be secured by still having opportunities to work or fix anything in technology world. 

In the computer science field, technical excellence is only as valuable as one's ability to communicate it. Throughout the program, I moved beyond just writing code to participating in the full Software Development Life Cycle (SDLC).  In my systems analysis coursework (CS 250/350), I learned to develop Software Design Documents (SDD). This taught me to communicate complex architectural decisions like why a specific design pattern was chosen to stakeholders who prioritize business outcomes over syntax. I have integrated peer code reviews into my workflow. By defending my logic and critiquing others', I’ve learned that "readable code" is a form of professional communication that ensures long-term project maintainability.

My approach to problem-solving has evolved from "making it work" to "making it optimal." I now view software engineering through the lens of Big O notation and resource constraints. During my advanced programming modules, I shifted from using basic nested loops to implementing more efficient structures like HashMaps or Balanced Trees to reduce time complexity. I’ve focused heavily on Unit Testing (JUnit). Testing for edge cases like null pointers or buffer overflows outside of the required assignments has made my engineering process more rigorous and my final products more "production-ready."

A system is only as strong as its weakest link. My work in databases and security has instilled a "Security by Design" mindset. I transitioned from using flat files for data storage to implementing Relational Databases (SQL). This wasn't just about storage; it was about ensuring Data Integrity through normalization (3NF) and preventing unauthorized access. In my security coursework (CS 405), I practiced identifying vulnerabilities like Buffer Overflows. This taught me to treat every user input as a potential threat, leading me to implement strict input validation and memory management techniques in all my C++ and Java projects.

The CS 360 Inventory App serves as a centerpiece of my portfolio, demonstrating my ability to design, develop, and optimize a full-stack mobile solution. This artifact highlights my proficiency in the Android Studio ecosystem and my ability to create user-centered applications that solve real-world logistical challenges. 

* Software Design and Engineering: The app follows a modular architecture using the Model-View-Controller (MVC) or Model-View-ViewModel (MVVM) pattern. By utilizing Android’s XML layout editor and Java/Kotlin back-end logic, I created a responsive UI with clear navigational flows between login, inventory grid, and item detail screens.
* Algorithms and Data Structures: To manage the inventory list efficiently, the app implements optimized data structures to handle CRUD (Create, Read, Update, Delete) operations. I focused on algorithmic efficiency for the search and sort functions, ensuring that even as the database grows, the user experience remains fast and fluid.
* Databases: The core of the application is a SQLite relational database. This artifact showcases my ability to design a schema with multiple tables specifically for user credentials and inventory items while ensuring data persistence across app sessions.
* Security and Notifications: Security is integrated through a secure User Authentication system, requiring valid credentials before accessing or modifying sensitive inventory data. Additionally, I implemented an automated notification system that triggers when stock levels reach zero, demonstrating my ability to integrate system-level services like SMS or Push Notifications based on specific logic triggers. 

This mobile application acts as a unifying thread for my technical skills. The Software Design aspect ensures a professional user experience; the Algorithms provide the speed necessary for warehouse environments; and the Database provides the secure, structured foundation required for reliable data management. Together, they represent my capacity to build complex, integrated systems from the ground up.
